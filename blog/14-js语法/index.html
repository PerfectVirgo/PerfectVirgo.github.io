1. JS 版本

  • ES 6 是最低要求

  • 历史版本

    ES 3，IE 6 支持，总体评价：垃圾

    ES 5，总体评价：还是垃圾

    ES 6，大部分浏览器支持，总体评价：一半垃圾一半好

    ES 2019 与 ES 6 差别不大

  • 为什么说 ES 6 一半垃圾

    因为 ES 不能删除以前的特性，要兼容旧网站

    也就是说以前能运行的网站，以后都要能运行

    对比 Python 3 你就能知道兼容的好处：稳定

  • 取其精华，去其糟粕

    对待 JS，这是我们的态度

2. 表达式与语句

  • 表达式

    1 + 2 表达式的值为 3

    add(1,2) 表达式的值为函数的返回值 （只有函数才有返回值）

    console.log 表达式的值为函数本身/函数体

    console.log(3) 表达式的值为多少？答：undefined

  • 语句

    var a = 1 是一个语句

  • 二者的区别

    表达式一般都有值，语句可能有也可能没有

    语句一般会改变环境（声明、赋值）

    上面两句话并不是绝对的

3. 大小写敏感

  • 不要写错

    var a 和 var A 是不同的

    object 和 Object 是不同的

    function 和 Function 是不同的

4. 空格

  • 大部分空格没有实际意义

    var a = 1 和 var a=1没有区别

    加回车大部分时候也不影响

    只有一个地方不能加回车，那就是 return 后面（如果你在 return 后面不加任何东西/加了回车，它会自动帮你补充一个 undefined）

5. 标识符

  • 规则

    第一个字符，可以是 Unicode 字母或 $ 或 _ 或中文

    后面的字符，除了上面所说，还可以有数字

    例如：var 9$，就会经典报错：Uncaught SyntaxError: Invalid or unexpected token，译为：没有捕获的语法错误：不合法的或者是没有被期待的字符串

  • 变量名是标识符

    var _ = 1

    var $ = 2

    var ______ = 6

    var 你好 = 'hi'

  • 其他标识符用到再说

8. if else 语句

  • 语法

    if (表达式) {语句1} else {语句2}

    {} 在语句只有一句的时候可以省略，不建议这样做

  • 变态情况

    • 表达式里可以非常变态，如 a = 1

    • 语句1里可以非常变态，如嵌套的 if else

      if (a < 100)
    if (a < 10)
        console.log('a小于10');

      结果是可以打印a小于10

    • 语句2里可以非常变态，如嵌套的 if else

      if (a < 100) {
        
} else 
    if (a > 10000) {
        console.log('a大于10000');
    }

      就变成了著名的 else if

      if (a < 100) {
        
} else if (a > 10000) {
    console.log('a大于10000');
} else {
        
}

      实际上这是两对儿 if…else，前面两个是一对儿，后面两个是一对儿，只不过它们的逻辑连起来正好跟它们的形式差不多

    • 缩进也可以很变态，如面试题常常下套

      a = 1
if (a === 2)
    console.log('a');
    console.log('a等于2');

      结果是打印a等于2，因为如果你省略了{}，那么 if 只会管到第一个语句

  • 最推荐使用的写法

    if (表达式) {
  语句
} else if (表达式) {
  语句
} else {
  语句
}

  • 次推荐使用的写法

    function fn(){
  if (表达式) {
      return 表达式
  }
  if (表达式) {
      return 表达式
  }
  return 表达式
}

9. switch 语句

  • 语法

    switch (fruit) {
  case "banana":
      // ...
      break;
  case "apple":
      // ...
      break;
  default:
      // ...
}

  • break

    大部分时候，省略 break 你就完了

    少部分时候，可以利用 break，如下

    a = 2
switch (a) {
  case 1:
  case 3:
      console.log('单数');
      break;
  case 2:
  case 4:
      console.log('双数');
      break;
}

    结果是打印双数

10. 问号冒号表达式

  • 语法

    表达式1 ? 表达式2 : 表达式3

  • 示例

    function max(a, b) {
  return a > b ? a : b;
}

    等价于

    function max(a, b) {
  if (a > b) {
      return a;
  } else {
      return b;
  }
}

11. && 短路逻辑

  • A && B && C && D 取第一个假值或 D

    并不会取 true / false

  • 示例

    window.f1 && console.log('f1存在')

    等价于

    if (window.f1) {
  console.log('f1存在')
}

12. || 短路逻辑

  • A || B || C || D 取第一个真值或 D

    并不会取 true / false

  • 示例

    a = a || 100

    等价于

    if (a) {
  a = a;
} else {
  a = 100;	//保底值
}

13. while 循环

  • 语法

    while (表达式) {语句}

    判断表达式的真假

    当表达式为真，执行语句，执行完再判断表达式的真假

    当表达式为假，执行后面的语句

  • 其他

    do…while 用得不多，自行了解

  • 变态示例

    var a = 0.1;
while (a !== 1) {
  console.log(a);
  a = a + 0.1;
}

    结果会造成死循环，原因是浮点数不精确造成的

14. for 循环

  • 语法糖

    for 是 while 循环的方便写法

    方便在哪？答：while 循环的四条语句每条都不能省略，否则会陷入死循环

  • 语法

    for (语句1;表达式2;语句3) {
  循环体
}

    先执行语句1

    然后判断表达式2

    如果为真，执行循环体，然后执行语句3

    如果为假，直接退出循环，执行后面的语句

  • 变态示例

    for (var i = 0; i < 5; i++) {
  setTimeout(()=>{
      console.log(i)
  },0)
}

    结果是打印5次5

    但如果把 var 改成 let ，居然会正常打印 0，1，2，3，4。原因是因为 for 循环在 let 的时候会有单独的逻辑，专门满足傻子的幻想

15. break 和 continue

  • 退出当前所有循环 V.S. 退出当前一次循环
  • 如果存在多重循环，不带参数的 break 语句和 continue 语句都只针对最内层循环。

16. label 语句

  • 语法

    foo: {
  console.log(1);
  break foo;
  console.log('本行不会输出');
}
console.log(2);

    foo 代表标识符

  • 面试示例

    {
  a: 1
}

    Q：上面的东西是什么？

    A：首先这个东西不是一个对象，它是一个代码块，里面有个标签 a，a 的语句就只有一个 1

    Q：如何把它变成一个对象？

    var b = {
  a: 1
}
